# 搭建文件系统
`BusyBox` 编译

```bash
wget https://busybox.net/downloads/busybox-1.36.0.tar.bz2
tar -jxvf busybox-1.36.0.tar.bz2 
```

<font style="background-color:#DF2A3F;">接着进图形化配置界面设置：勾选 </font>`<font style="background-color:#DF2A3F;">Settings</font>`<font style="background-color:#DF2A3F;"> ---> Build static binary file (no shared lib) </font>

然后编译

```bash
make -j$(nproc)
make install
```

这里高版本的 linux 内核 6.x 版本会报错：`error: ‘TCA_CBQ_MAX’ undeclared`

> 根本原因： BusyBox 1.36.0 中的 networking/tc.c 使用了 CBQ (Class Based Queueing) 调度算法的相关定义。然而，在 Linux 内核 6.8 及更高版本中，CBQ 调度算法因为过时且无人维护，已经被完全移除了
>

解决办法：不用相关的工具 `tc`

进 busybox 的图形化配置界面：`Networking Utilities`--->找到 tc 并取消勾选： [ ] tc (13 kb)

然后编译就可以了

# 配置文件系统
![](/images/25b31db50fb33d656544ee3cd5a736a9.png)

在 `_install` 目录下创建基本的文件系统结构：

```bash
cd _install
mkdir -pv {bin,sbin,etc,proc,sys,dev,home/ctf,root,tmp,lib64,lib/x86_64-linux-gnu,usr/{bin,sbin}}
touch etc/inittab
mkdir etc/init.d
touch etc/init.d/rcS
chmod +x ./etc/init.d/rcS
```

在创建的 `./etc/inittab` 中写入内容：

```bash
::sysinit:/etc/init.d/rcS
::askfirst:/bin/login
::ctrlaltdel:/sbin/reboot
::shutdown:/sbin/swapoff -a
::shutdown:/bin/umount -a -r
::restart:/sbin/init
```

接着配置上面指定的系统初始化脚本`etc/init.d/rcS`：

> 主要是挂载各种文件系统，以及设置各目录的权限，并创建一个非特权用户
>

```bash
#!/bin/sh
chown -R root:root /
chmod 700 /root
chown -R ctf:ctf /home/ctf

mount -t proc none /proc
mount -t sysfs none /sys
mount -t tmpfs tmpfs /tmp
mkdir /dev/pts
mount -t devpts devpts /dev/pts

echo 1 > /proc/sys/kernel/dmesg_restrict
echo 1 > /proc/sys/kernel/kptr_restrict

echo -e "\nBoot took $(cut -d' ' -f1 /proc/uptime) seconds\n"

cd /home/ctf
su ctf -c sh

poweroff -d 0  -f
```

```bash
chmod +x ./etc/init.d/rcS
```

配置用户组相关权限：

```bash
echo "root:x:0:0:root:/root:/bin/sh" > etc/passwd
echo "ctf:x:1000:1000:ctf:/home/ctf:/bin/sh" >> etc/passwd
echo "root:x:0:" > etc/group
echo "ctf:x:1000:" >> etc/group
echo "none /dev/pts devpts gid=5,mode=620 0 0" > etc/fstab
```

# 打包文件系统
## QCOW2 格式
QEMU Copy-on-Write version 2 `QCOW2` 是 QEMU 的一种常用的硬盘镜像格式

```bash
qemu-img create -f qcow2 rootfs.qcow2 50M
```

然后就可以把这个镜像文件挂载称网络块设备：

> 但是需要先手动启用 nbd 内核模块
>

```bash
sudo modprobe nbd
```

> NBD（Network Block Device，网络块设备）内核模块的作用是：让 Linux 系统能够把远程计算机上的磁盘空间或文件，通过网络映射成一个本地的“物理磁盘”（块设备）
>

```bash
sudo qemu-nbd -c /dev/nbd0 ./rootfs.qcow2
```

然后就是把它按照对应的文件系统格式化

```bash
sudo mkfs.ext4 /dev/nbd0
```

挂载：

```bash
sudo mount /dev/nbd0 /mnt
```

拷贝配置好的文件系统：

```bash
sudo cp -auv _install/* /mnt
sudo chown -R root:root /mnt/
sudo chmod 700 /mnt/root
sudo chown -R 1000:1000 /mnt/home/ctf/
```

最后卸载并且解绑 nbd 就可以了：

```bash
sudo umount /mnt
sync
sudo qemu-nbd -d /dev/nbd0
```

## ext4 格式
首先创建空白 ext4 镜像文件，这里 bs 表示块大小，count 表示块的数量：

```bash
dd if=/dev/zero of=rootfs.img bs=1M count=32
```

格式化为 ext4 格式：

```bash
mkfs.ext4 rootfs.img
```

拷贝文件：

```bash
mkdir tmp
sudo mount rootfs.img ./tmp/
sudo cp -rfp _install/* ./tmp/
sudo chown -R root:root ./tmp/
sudo chmod 700 ./tmp/root
sudo chown -R 1000:1000 ./tmp/home/ctf/
sudo umount ./tmp
```

## cpio 格式
在 `_install` 目录下使用如下命令打包文件系统为 cpio 格式：

```bash
find . | cpio -o --format=newc > ../rootfs.cpio
#或者find . | cpio -o -H newc > ../rootfs.cpio
```

<font style="color:rgba(0, 0, 0, 0.87);">还可以重新解包文件系统：</font>

```bash
cpio -idv < ./rootfs.cpio
```

如果题目给了 cpio 文件，就可以解包然后修改完再重新打包

# 启动内核
捣鼓了这么久总算是可以内核！启动！）

```bash
#!/bin/sh
qemu-system-x86_64 \
    -m 128M \
    -kernel ./bzImage \
    -hda ./rootfs.qcow2 \
    -monitor /dev/null \
    -append "root=/dev/sda rw rdinit=/sbin/init console=ttyS0 oops=panic panic=1 nokaslr" \
    -cpu kvm64,+smep \
    -smp cores=2,threads=1 \
    -nographic \
    -snapshot \
    -s
```

要注意不同文件系统格式在参数上的不同：

ext4：

```bash
-hda  ./rootfs.img \
```

cpio：

```bash
-initrd  ./rootfs.cpio \
-append "root=/dev/ram..."
```

# 调试内核
一般的 kernel 题目都是给了镜像的，可以通过在 qemu 启动的时候加上 gdb 参数`-s`（等于`-gdb tcp::1234` ），然后就可以利用 `vmlinux` 文件对内核进行动调了

```bash
gdb ./vmlinux
target remote :1234
```

或者

```bash
gdb -q -ex "target remote localhost:1234"
```

例子：断点系统调用

```bash
b __x64_sys_execve
```
