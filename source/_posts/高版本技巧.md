## 泄漏 libc 地址
bss 段上有放着 stdin、stdout 或者 stderr 的指针的话，有条件的话也可以从这里下手去泄漏

实际上 bss 段上存着的是指向 libc 当中 _IO_2_1_stdin 之类的结构体的位置，

这些指针不在 bss 段的话就也在 libc 里面（具体看编译器是怎么处理的）

## ROP 利用点
binary中有syscall的话可以优先考虑SROP

其中没有控制rax的gadget可以先利用read的返回值来控制rax再来syscall



系统调用**dup**：

```plain
int dup(int oldfd)
```

复制oldfd对应的fd到一个新的文件描述符，这两个fd都指向同一个文件

系统会分配当前可用的最小的fd，返回值就是新的fd

系统调用**dup2**：

```plain
int dup2(int oldfd, int newfd)
```

可以指定复制到的fd，如果指定fd已经被打开会自动关闭

dup2用法：在沙盒限制read的fd必须为0时可以把标准输入fd 0用来跟flag文件的fd交换

返回时rax=0，binary有syscall_ret，考虑srop

1. 先利用read读取15个字节控制rax=15
2. syscall调用sigreturn，sigframe.rax=0，调用read写shellcode到bss段

同时控制rsp到payload-8，准备进行栈迁移

1. payload=p64(栈溢出函数地址)+p64(shellcode地址)+shellcode写进去
2. 再溢出一遍，利用read读取15个字节控制rax=15
3. syscall调用sigreturn，sigframe.rax=10，调用mprotect改bss段权限可执行，

同时控制rsp到payload，栈迁移执行shellcode
