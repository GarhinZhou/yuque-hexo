### 允许非Root用户执行docker命令
添加docker用户组：

```plain
sudo groupadd docker
```

将当前用户添加到用户组：

```plain
sudo usermod -aG docker $USER
```

使权限生效：

```plain
newgrp docker
```

### Docker指令
#### 镜像相关
```bash
# 拉取镜像到本地
docker pull 镜像名称[:tag]

# 查看本机上的所有镜像
docker images

# 删除本地镜像
docker rmi 镜像名称[:tag]

# 导出本地镜像
docker save -o 导出的路径 镜像名称[:tag]

# 加载本地的镜像文件
docker load -i 镜像文件
```

#### 容器相关
```bash
# 容器构建
docker build -t image .

# 运行容器
docker run -d -p 宿主机端口:容器端口 --name 容器名称 镜像名称[:tag]
# -d：代表后台运行容器
# -p 宿主机端口：容器端口 ：为了映射当前主机的端口和容器的端口
# --name：容器名称；指定容器的名称

# 进入容器内部
docker exec -it 容器id bash

# 停止容器
docker stop 容器id

# 强制停止容器
docker kill 容器id

# 删除容器
docker rm 容器id

# 查看所有容器
docker ps
```

容器加载

```python
docker load -i imi_mqtt_pwn_v3.tar
docker run --rm -it -p 9999:9999 imi_mqtt:latest
```

### Dockerfile指令
```dockerfile
FROM ubuntu:22.04 #获取镜像

COPY ./hostpath /containerpath #复制文件到容器内

RUN command #build时执行的指令

CMD ["command"] #容器启动时执行的指令
#在Dockerfile中可以存在多个CMD指令，但仅最后一个会生效

EXPOSE port #设置暴露端口，可以设置多个

WORKDIR /workpath #设置工作目录
```

## Binary编译相关
ASLR和PIE的关系：ASLR是能力，PIE是支持能力

PIE只是在编译的过程中赋予了ELF加载到内存时其加载基址随机化的功能，也就是说PIE编译出来的ELF如果在ASLR=0的情况下，ELF的加载基址也是不会变的

ASLR刚开始设计的时候是作为操作系统功能提供的，只考虑了当时技术背景下executable加载后stack、heap、libraries的随机化功能，因而有多种绕过方式，这一时期的ASLR的定义也就成了“对stack、heap、libraries的随机化”。后来设计者就考虑executable的加载基址随机化以使得bypass失效，然而这一点在技术细节上需要编译器来实现，因此gcc开始支持PIE选项，使得编译出来的executable像是一个特殊的so，可以被操作系统加载到随机化的内存地址

### Gcc
指定 libc 版本编译：

但是实际上还是跟 gcc 的版本有关系，高版本的 gcc 编译出来仍然需要是 glibc2.34 版本往上

```c
gcc -o pwn pwn.c -L/libcpath -Wl,--rpath=/libcpath -Wl,--dynamic-linker=/libcpath/ld
```

1. 关闭NX（堆栈不可执行）

```bash
gcc -z execstack -o 编译完的文件名 待编译的文件名
```

2. 关闭Canary（栈溢出保护）

```bash
gcc -fno-stack-protector -o 编译完的文件名 待编译的文件名
```

3. 关闭程序PIE（程序地址随机化保护）

```bash
gcc -no-pie 编译完的文件名 待编译的文件名
```

4. 关闭linux系统ASLR

```bash
sudo -s 
echo 0 > /proc/sys/kernel/randomize_va_space
exit
```

5. 开启linux系统ASLR

```bash
sudo -s 
echo 2 > /proc/sys/kernel/randomize_va_space
```

6. 64位linux下面编译出32位可执行程序

```bash
gcc -m32 -z execstack -fno-stack-protector -o 编译完的文件名 待编译的文件名
```

-后面接的都是参数，参数顺序可以换

-o得在文件名前面，例如：

关canary、PIE和NX的编译选项：

```bash
gcc -fno-stack-protector -fno-pie -no-pie -z execstack -o ret2text ret2text.c
```

7. 设置 got 表的权限：

```rust
-z norelro // 关闭
-z lazy // 部分开启
-z now // 全部开启
```
