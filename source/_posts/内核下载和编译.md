内核主要有以下几种类别

+ Prepatch (RC) ：主线内核的预发布版本，包含了最新的待测试的内核特性，由 Linus Torvalds 维护
+ Mainline：主线内核版本，RC 版本的新特性经过测试后便会合并到主线，每 9～10 周发一个版本，由 Linus Torvalds 维护
+ Stable：主线内核发布后便会变为 Stable 状态，其仅会被 stable kernel 维护者从主线树后向移植一些漏洞修复，直到下个内核版本释出。Stable kernel 根据需要进行更新，通常是一周一次
+ Longterm：部分内核版本会被选作长期支持版（LTS），相比起 Stable 内核有着更久的支持时长，通常仅会被后向移植重要的漏洞修复，且更新周期较慢（尤其是对于更老的版本）

# 下载源码
清华源：

```bash
wget https://mirrors.tuna.tsinghua.edu.cn/kernel/v6.x/linux-6.12.16.tar.xz
unxz ./linux-6.12.16.tar.xz
```

# 验证内核 tar 包签名
导入 <font style="color:rgba(0, 0, 0, 0.87);">Linus Torvalds 和 Greg Kroah-Hartman 的公钥：</font>

```bash
gpg2 --locate-keys torvalds@kernel.org gregkh@kernel.org
```

<font style="color:rgba(0, 0, 0, 0.87);">导入不了就手动导入：</font>

```python
gpg2 --keyserver hkps://keyserver.ubuntu.com --recv-keys 38DBBDC86092693E
```

<font style="color:rgba(0, 0, 0, 0.87);">从清华大学镜像站下载内核签名进行校验：</font>

```bash
wget https://mirrors.tuna.tsinghua.edu.cn/kernel/v6.x/linux-6.12.16.tar.sign
gpg2 --verify linux-6.12.16.tar.sign
```

输入最后一句之后结果是：

```bash
gpg: assuming signed data in 'linux-6.12.16.tar'
gpg: Signature made Sat Feb 22 00:02:55 2025 AEDT
gpg:                using RSA key 647F28654894E3BD457199BE38DBBDC86092693E
gpg: Good signature from "Greg Kroah-Hartman <gregkh@kernel.org>" [unknown]
gpg: WARNING: This key is not certified with a trusted signature!
gpg:          There is no indication that the signature belongs to the owner.
Primary key fingerprint: 647F 2865 4894 E3BD 4571  99BE 38DB BDC8 6092 693E
```

因为导入的公钥没有可信的签名，无法证明其确实来自于 Linus Torvalds 和 Greg Kroah-Hartman

为了消除这个问题，可以选择使用 TOFU 信任对应的密钥：

```bash
gpg2 --tofu-policy good 38DBBDC86092693E
```

重新验证内核签名：

```bash
gpg2 --trust-model tofu --verify ./linux-6.12.16.tar.sign
```

通过之后就可以解压缩得到内核源码：

```bash
tar -xf linux-6.12.16.tar
```

问了一下 Gemini：

直接复制了 Ubuntu 系统的配置文件（/boot/config-...），而在这个配置中，CONFIG_SYSTEM_TRUSTED_KEYS 指向了一个只有 Ubuntu 官方打包机上才有的路径：debian/canonical-certs.pem。本地没有这个证书文件，所以 make 报错停止

解决方法：

```bash
# 1. 禁用受信任的系统证书路径
scripts/config --disable SYSTEM_TRUSTED_KEYS

# 2. 禁用吊销证书路径
scripts/config --disable SYSTEM_REVOCATION_KEYS

# 3. 将配置项设置为空字符串（双重保险）
scripts/config --set-str CONFIG_SYSTEM_TRUSTED_KEYS ""
scripts/config --set-str CONFIG_SYSTEM_REVOCATION_KEYS ""
```
